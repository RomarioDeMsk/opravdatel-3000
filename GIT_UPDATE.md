# Как добавить новую версию на Git

## Быстрая инструкция

### 1. Проверьте статус изменений

```bash
git status
```

Покажет все измененные, добавленные и удаленные файлы.

### 2. Добавьте все изменения

```bash
# Добавить все изменения
git add .

# Или добавить конкретные файлы
git add index.html style.css modules/ui.js
```

### 3. Сделайте коммит с описанием

```bash
git commit -m "Описание изменений"
```

**Примеры хороших сообщений коммита:**
```bash
git commit -m "Добавлена система голосования с анимацией swipe"
git commit -m "Обновлены шаблоны отговорок, уменьшено до 1000 вариантов"
git commit -m "Интегрирована панель голосования в окно генератора"
git commit -m "Исправлены баги и улучшен UI"
```

### 4. Загрузите на GitHub

```bash
git push origin main
```

Если используете ветку `master`:
```bash
git push origin master
```

## Если возникли проблемы

### Ошибка: "Your branch is ahead of 'origin/main'"
**Решение:** Просто выполните `git push origin main`

### Ошибка: "Please tell me who you are"
**Решение:** Настройте Git:
```bash
git config --global user.name "Ваше Имя"
git config --global user.email "ваш@email.com"
```

### Ошибка: "Updates were rejected"
**Решение:** Сначала получите последние изменения:
```bash
git pull origin main
# Разрешите конфликты если есть
git push origin main
```

### Ошибка: "Permission denied"
**Решение:** 
- Проверьте правильность URL репозитория
- Убедитесь, что вы авторизованы на GitHub
- Возможно, нужно использовать токен доступа вместо пароля

## Полезные команды

### Посмотреть историю коммитов
```bash
git log
```

### Посмотреть изменения в файлах
```bash
git diff
```

### Отменить изменения (до коммита)
```bash
git restore .
# Или для конкретного файла
git restore index.html
```

### Отменить последний коммит (но сохранить изменения)
```bash
git reset --soft HEAD~1
```

### Создать тег версии
```bash
git tag -a v1.0.0 -m "Версия 1.0.0"
git push origin v1.0.0
```

---

## Рекомендации

1. **Делайте коммиты часто** — после каждого значимого изменения
2. **Пишите понятные сообщения** — что именно изменилось
3. **Проверяйте статус** — `git status` перед коммитом
4. **Тестируйте перед пушем** — убедитесь, что все работает
